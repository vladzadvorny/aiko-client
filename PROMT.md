Ты — senior frontend-разработчик, специализирующийся на React. Создай раздел в файле src/pages/Chat.js чата AI-ассистента в стиле Character.AI.

### Требования к приложению:

1. **Стек**: Используй valtio для стейт менеджмента. в нём храни только данные. а имитацию запросов в файле
2. **Структура проекта**:
   - в файле `src/pages/Chat.js` сам чат
   - в файле `src/components/LeftSidebar.js` список диалогов. при клике на каждый он открывается в файле `src/pages/Chat.js`

3. **Функционал**:
   - **Главная страница (Home)**:
     - Создай файл `src/pages/Home.js` он доступен по адресу `/`. Стили в файле `src/pages/Home.scss`. Эти файлы писать по аналогии с `src/pages/Auth.js` и `src/pages/Auth.scss`
     - Сетка карточек популярных персонажей (имя, аватар, короткое описание, теги: "Фэнтези", "Помощник", "Юмор" и т.д.)
     - Поиск по имени/тегу
     - Фильтры по категориям
     - Кнопка "Создать своего персонажа" (модальное окно с формой: имя, описание, аватар, стиль общения)
   - **Страница чата (Chat)**:
     - Сайдбар с историей чатов (с моковыми данными) в файле `src/components/LeftSidebar.js`
     - Основное окно чата:
       - Сообщения от пользователя и AI (с аватарками)
       - Анимация "печатает..." при "ответе" AI
       - Поле ввода с кнопкой отправки
       - Автоскролл к последнему сообщению
     - Кнопка "Регенерировать ответ", "Очистить чат", "Поделиться"
   - **Мобильная адаптивность** (mobile-first подход)

4. **Моковые данные** (в `mocks/`):
   - Минимум 6 разных персонажей с:
     ```ts
     interface Character {
       id: string
       name: string
       avatar: string // URL или base64
       description: string
       tags: string[]
       greeting: string
       personality: string
     }
     ```
   - Моковая история чатов (3–5 диалогов с разной длиной)
   - Имитация задержки ответа AI (setTimeout 1–2 сек)

5. **Состояние**:
   - Используй valtio для управления:
     - Списком персонажей
     - Текущим чатом
     - Историей сообщений
     - Выбранным персонажем

6. **Дополнительно**:
   - Плавные анимации (Framer Motion опционально)
   - Обработка ошибок ввода (пустое сообщение)
   - Локальное сохранение чатов в localStorage

7. **Выходной формат**:
   - Используй **функциональные компоненты**, **хуки**, и пиши код в таком же стиле, как и в других файлах
   - Комментарии в коде на английском (best practices)
   - Моки на русском, а картинки используй со стоков
